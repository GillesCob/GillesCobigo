## Pourquoi JSON ne suffit pas pour envoyer un fichier

Un fichier n'est pas une donnée texte. JSON ne sait pas encoder des données binaires. Pour envoyer un PDF vers un backend Express, il faut utiliser `FormData` - un format multipart/form-data qui emballe le fichier binaire et les métadonnées dans la même requête.

Côté frontend, la construction est simple :

```ts
const formData = new FormData();
formData.append("file", file);
formData.append("propertyId", propertyId);
formData.append("documentType", "BLUEPRINT");

const response = await apiClient.post("/api/documents", formData, {
  headers: { "Content-Type": "multipart/form-data" },
});
```

## Multer reçoit le fichier côté backend

Express ne sait pas lire le multipart nativement. Multer est le middleware qui intercepte la requête, lit le fichier et l'expose sur `req.file`. La configuration `memoryStorage` garde le fichier en mémoire sous forme de Buffer - pas de fichier temporaire sur le disque, idéal pour un transfert direct vers Supabase.

Le fichier est alors accessible via `req.file.buffer`, `req.file.originalname` et `req.file.mimetype`.

## Supabase Storage reçoit le Buffer

Le client Supabase Storage prend le Buffer directement. On génère un nom unique avec `Date.now()` pour éviter les collisions, on précise le `contentType` pour que le navigateur sache comment ouvrir le fichier, et Supabase retourne le chemin du fichier dans le bucket.

Ce chemin est ensuite sauvegardé en base de données via Prisma. C'est lui qui permettra de reconstruire l'URL publique côté frontend.

```ts
const uniqueFileName = `${Date.now()}-${fileName}`;
const { data, error } = await supabase.storage
  .from("documents")
  .upload(uniqueFileName, buffer, { contentType: mimeType });
```

## Reconstruire l'URL publique côté frontend

Supabase Storage expose les fichiers d'un bucket public via une URL prévisible. On stocke uniquement le nom du fichier en base, et on reconstruit l'URL complète côté frontend depuis une variable d'environnement. Pas d'URL stockée en dur, pas de couplage fort avec l'infrastructure.

```tsx
href={`${import.meta.env.VITE_SUPABASE_URL}/storage/v1/object/public/documents/${doc.url}`}
```

La chaîne complète : `FormData` > `multipart/form-data` > `multer` > `req.file.buffer` > Supabase Storage > chemin en BDD > URL reconstruite. Chaque couche a une responsabilité claire.
