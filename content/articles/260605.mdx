## Préambule

Ok on y est !
Comme vu dans les précédents articles, notre objectif dans les prochains jours va être de disséquer ce qu’a pu nous créer Claude Code dans le cadre du projet ChouxFleurs2.
Nous ne souhaitons pas prendre pour acquis ce qu’il a pu générer mais plutôt analyser le rendu, identifier les éléments qui ne nous conviennent pas et les corriger. Il existe toujours une multitude de chemins possibles pour obtenir un résultat visuellement identique. Nous souhaitons cependant obtenir un code propre, maintenable, scalable et sécurisé.

## Analyser le résultat obtenu

Nous avions un prompt assez basique lors de la création de notre app côté Front : Utiliser Vite, React et c’est plus ou moins tout.
Nous obtenons finalement une structure classique par type : tous les components, hooks ou pages sont regroupés dans des dossiers distincts. Je remarque plusieurs points :

- Pas de Zustand.
- Pas de Tanstack Query.
- Pas d’Axios.
- Pas de JWT.
- Utilisation de Context pour la gestion d'état.
- Présence de Radix UI.

Je note quelques ajustements nécessaires : placer le `BrowserRouter` dans le `main.tsx` et non dans `App`, renommer les interfaces avec un préfixe "I" (ex: `IUser`), et mieux appliquer les principes SOLID en déplaçant la logique métier hors des composants de rendu.

## Zustand

Zustand est une bibliothèque qui va permettre de rendre accessible dans l’ensemble de mon code les états ainsi que les données de mon application, évitant ainsi le _Prop Drilling_.
Bien que le `Context` soit utilisé actuellement, Zustand a l'avantage de ne pas re-rendre l’ensemble des composants lors de la modification d’un état, d'être plus simple à configurer et accessible en dehors des composants React. Nous transformerons donc notre code pour l'adopter.

## Tanstack Query

Tanstack Query est la bibliothèque de référence pour simplifier et optimiser les requêtes réseau.
Elle permet une gestion native du cache et des états de chargement (`isLoading`, `isError`, etc.), remplaçant ainsi avantageusement de multiples `useState`. Sa mise en place sera un levier majeur pour professionnaliser le projet.

## Radix et SHADCN

L'IA m'a fait découvrir de nouveaux outils :

- **Radix UI** : Une bibliothèque de primitives bas niveau. Les composants sont fonctionnels et accessibles, mais sans aucun style prédéfini.
- **SHADCN** : Utilise les primitives de Radix en y ajoutant des éléments de design. Cela permet d'obtenir rapidement des composants complexes et esthétiques, tout en gardant une totale liberté pour les personnaliser dans le dossier `components`.

## En déduire les règles de conception

L'objectif est d'utiliser les outils les plus efficaces et professionnels.
À partir de ces constats, j'ai mis à jour le fichier **CLAUDE.md** de mon projet. Ce fichier agit comme le « cerveau » du projet, dictant à l’IA les standards techniques à respecter dès la première ligne de code. Nous ferons évoluer ces règles au fil de notre progression pour garantir la cohérence de mes futurs développements.
